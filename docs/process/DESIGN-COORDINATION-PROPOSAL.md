## 1. План аудита и потенциальных изменений (Фаза 1 ПП)

### Что я буду проверять:
1. **Цветовые токены** — все ли компоненты используют CSS-переменные, нет ли хардкода
2. **Состояния элементов** — hover, focus, disabled, error, loading у кнопок, полей, селектов, таблиц
3. **Обводки/границы** — есть ли у всех элементов, которые должны иметь border, видимая граница в покое
4. **Тёмная тема** — нет ли зелёного отлива, коричневых артефактов
5. **Светлая тема** — нет ли тёмных пятен,sidebar корректно светлый
6. **Контраст** — WCAG AA 4.5:1 для всех текстовых пар
7. **Анимации** — нет ли тяжёлых box-shadow, нет ли дёрганий при смене состояний
8. **Адаптивность** — Desktop 1440px, Tablet 768px, Mobile 375px

### Что я потенциально буду менять:
- Унификация хардкод-цветов на CSS-переменные
- Добавление недостающих состояний (hover/focus/disabled) для компонентов
- Исправление контрастных пар ниже WCAG AA
- Оптимизация тяжёлых теней и transition
- Приведение обводок к единой системе (1px, `--border` токен)

## 2. Цветовые токены и стилевые правила (текущее состояние + план)

### Текущая система (v3 — "Neutral Slate + Deep Violet"):

| Токен | Светлая | Тёмная | Tailwind |
|-------|---------|--------|----------|
| Background | `#FBFBFD` | `#0B0E14` | `bg-background` |
| Primary | `#7C3AED` | `#A78BFA` | `bg-primary` |
| Border | `#E4E7EC` | `#2A2E38` | `border-border` |
| Muted | `#F4F5F8` | `#14171F` | `bg-muted` |
| Sidebar | `#FFFFFF` | `#0E1119` | `bg-sidebar` |
| Glass | `rgba(255,255,255,0.78)` | `rgba(14,17,25,0.72)` | `.glass-surface` |

### Что я планирую ввести/скорректировать:

**Потенциальные НОВЫЕ токены:**
- `--color-focus-ring` — единый цвет обводки фокуса (сейчас `--ring` = primary, что OK)
- `--color-hover-overlay` — единый полупрозрачный слой для hover-состояний (сейчас используется `--muted` вручную)

**Потенциальные корректировки:**
- Проверить, что ВСЕ hover-состояния используют `bg-[var(--muted)]` а не хардкод
- Проверить, что ВСЕ focus-состояния используют `ring-[var(--ring)]` а не кастомные
- Убедиться что обводки везде `border-[var(--border)]` (1px)

**Семантические токены (проверю использование):**
- `--success`, `--warning`, `--destructive`, `--info` — используются ли системно через `<Badge>`, `<Alert>`, `<StatusBadge>`, или переопределены локально?

## 3. Границы моей ответственности

### Я НЕ трогаю:
- **Сетку/шрифты/отступы** — typography system не трогаю (Font Geist Sans/Mono зафиксированы)
- **API/логику** — ни одного серверного файла, ни одного route handler
- **Prisma schema/migrations** — данные вне scope
- **Анимации производительности** — если не будет конкретного бага с lag
- **Компоненты данных** — Table, GanttChart, ProposalEditor — только визуальные стили, не логику
- **PDF генерацию** — Tier B, frozen API
- **JWT/auth** — Tier A, frozen

### Я ТОЛЬКО проверяю и правлю:
- CSS-переменные и их использование
- Tailwind-классы цветов/границ/теней
- Состояния UI-компонентов (hover/focus/disabled)
- Контрастность текста
- Адаптивность цветов при resize
