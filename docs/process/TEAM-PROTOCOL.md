## 2. Процесс работы

### Фаза 1: Планирование (Buffy)
1. Buffy анализирует задачу, читает v4.0 аналоги, пишет спецификацию
2. Buffy обновляет BUSINESS-LOGIC.md если нужно
3. Buffy пишет задачу в AI_COLLABORATION.md
4. Buffy помечает задачу как [NEW]

### Фаза 2: Исполнение (MiMo)
1. MiMo читает AI_COLLABORATION.md, находит задачу [NEW]
2. MiMo читает BUSINESS-LOGIC.md, SPEC-ДОРАБОТКА.md
3. MiMo помечает задачу как [IN_PROGRESS]
4. MiMo реализует по спецификации
5. MiMo запускает `npx tsc --noEmit` — исправляет ошибки
6. MiMo пишет отчёт о выполненном
7. MiMo помечает задачу как [DONE]

### Фаза 3: Проверка (Buffy)
1. Buffy читает отчёт MiMo
2. Buffy проверяет код на соответствие спецификации
3. Buffy запускает проверку типов
4. Buffy либо принимает [VERIFIED], либо пишет замечания [REWORK]
5. Если [REWORK] — MiMo исправляет

### Фаза 4: Интеграция (Buffy)
1. Buffy проверяет что модуль не сломал соседние
2. Buffy обновляет ЧЕК-ЛИСТ-МИГРАЦИИ.md
3. Buffy запускает тестирование в браузере

## 4. Стандарты кода

### Next.js 16 / App Router
- `'use client'` — для клиентских компонентов
- `'use server'` — для серверных
- Route Handlers: `export async function GET/POST/PUT/DELETE`
- Динамические роуты: `[id]/page.tsx`

### UI
- Tailwind CSS (v4)
- lucide-react для иконок
- Компоненты из `src/components/ui/`
- Цвета через CSS переменные: `var(--primary)`, `var(--muted-foreground)` и т.д.

### Состояние
- `useState` / `useCallback` / `useEffect` для локального состояния
- Zustand stores в `src/stores/` для глобального

### API
- Возвращать `{ success: boolean, data: T, message?: string }`
- HTTP статусы: 200, 201, 400, 401, 404, 500
- Аутентификация через `apiResponse` и `getAuthUser()`

## 5. Качество

### Минимальные требования к каждому модулю:
- [ ] TypeScript `tsc --noEmit` — 0 ошибок
- [ ] Все импорты используются (нет dead code)
- [ ] Loading state (спиннер/скелетон при загрузке)
- [ ] Empty state (сообщение когда данных нет)
- [ ] Error state (обработка ошибок API)
- [ ] Edge cases (пустая корзина, 0 товаров, удаление последнего, и т.д.)
- [ ] Адаптивность (мобильная версия не ломается)
- [ ] Нет закомментированного кода `console.log`
- [ ] Соответствие BUSINESS-LOGIC.md

Если хоть один пункт не выполнен — задача возвращается [REWORK].
