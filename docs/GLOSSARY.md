# docs/GLOSSARY.md — Глоссарий терминов kppdf-5.0

> Цель: единый справочник терминов с однозначными определениями, чтобы устранить разночтения между агентами, разработчиками, и бизнес-аналитиками. Если термин ниже — читай это определение; не додумывай.

**Дата**: 2026-06-20 (создан в Cycle 41, docs-cleanup)
**Аудитория**: AI-агенты, разработчики, владелец продукта.

---

## 🏢 Бизнес-домен (workflow)

| Термин | Определение | EN alias | Где в коде/models |
|--------|-------------|----------|------------------|
| **КП** | Коммерческое предложение. Документ, который формируется из витрины товаров + шаблона, отправляется клиенту, может быть конвертирован в договор. | Proposal | `prisma.Proposal`, `src/app/(dashboard)/proposals/` |
| **Договор** | Юридический документ, фиксирующий обязательства сторон. Конвертируется из КП после согласования. Содержит позиции, сумму, статус оплаты. | Contract | `prisma.Contract`, `src/app/(dashboard)/contracts/` |
| **Производственный заказ** | Заказ на изготовление. Создаётся автоматически из договора при статусе «оплачено» или вручную. Содержит задачи, привязанные к рабочим центрам. | Production Order | `prisma.ProductionOrder`, `src/app/(dashboard)/production/` |
| **Гантт** | Диаграмма задач производственного заказа (день/неделя/месяц). Drag&drop для сдвига сроков. | Gantt chart | `src/app/(dashboard)/production/gantt` |
| **Снабжение** | Процесс закупки материалов для производства. Авто-список материалов из модулей товара → заявки на закупку → заказы поставщикам → входящие накладные. | Procurement | `src/app/(dashboard)/warehouse/purchase-requests/` |
| **Заявка на закупку** | Документ, фиксирующий потребность в материале (qty, срок, поставщик). Может быть конвертирована в заказ поставщику. | Purchase Request | `prisma.PurchaseRequest` |
| **Заказ поставщику** | Подтверждённый документ закупки (после заявки). Исполняется поставщиком → входящая накладная. | Supplier Order | `prisma.SupplierOrder` |
| **Входящая накладная** | Документ прихода от поставщика. Обновляет остатки на складе. | Incoming Invoice | `prisma.IncomingInvoice` |
| **Отгрузка** | Передача готового товара клиенту. Фиксируется в shipment, уменьшает остатки. | Shipment | `prisma.Shipment`, `src/app/(dashboard)/warehouse/shipping/` |
| **Закрытие заказа** | Финальный этап: акт выполненных работ + финансовое закрытие. Заполняет акт сверки, формирует финансовые документы. | Order Closing | `prisma.OrderClosing` |
| **Акт сверки** | Документ сверки взаиморасчётов между компанией и клиентом/поставщиком. | Reconciliation Act | `prisma.ReconciliationAct` |
| **Тендер** | Процедура выбора поставщика через конкурс. | Tender | `prisma.Tender`, `src/app/(dashboard)/admin/tenders/` |
| **РНР** | Реестр нормативной документации (для сертификации продукции). | RNR Registry | `prisma.RppEntry` |

## 🧩 Товарная иерархия

| Термин | Определение | Где в коде |
|--------|-------------|-----------|
| **Категория товара** | Логическая группировка товаров. | `prisma.ProductCategory` |
| **Товар** | Единица продажи. Содержит фотографии, цену, модули. | `prisma.Product` |
| **Модуль товара** | Логический блок внутри товара (например, «дверь», «коробка», «замок»). Содержит список работ + материалов. | `prisma.ProductModule` |
| **Вид работ модуля** | Конкретная операция, которую нужно выполнить для изготовления модуля (например, «сварка каркаса», «зачистка швов»). | `prisma.ModuleWorkType` |
| **Материал модуля** | Конкретный ресурс (металл, крепёж, краска), который нужен для модуля. | `prisma.ModuleMaterial` |

## 🏭 Производственные сущности

| Термин | Определение | Где в коде |
|--------|-------------|-----------|
| **Рабочий центр** | Подразделение производства (цех, линия, пост), выполняющее определённые виды работ. | `prisma.WorkCenter` |
| **Вид работ** | Тип производственной операции (сварка, гибка, покраска). | `prisma.WorkType` |
| **Рабочий (сотрудник)** | Человек, привязанный к рабочему центру, выполняющий задачи. | `prisma.Worker` |
| **Задача** | Конкретная единица работы в производственном заказе (например, «сварить каркас двери №3»). | `prisma.OrderTask` |

## 📄 Документ-шаблоны

| Термин | Определение | Где в коде |
|--------|-------------|-----------|
| **Шаблон документа** | Переиспользуемая скелетная структура документа (КП/договор/счёт). Содержит блоки (текст/таблица/разделитель) + задний фон. | `prisma.DocumentTemplate` |
| **A4 Canvas** | WYSIWYG-редактор шаблона с drag-and-drop блоков на «листе A4». | `src/components/ui/a4-canvas/` |
| **Блок документа** | Элемент внутри шаблона: текст с колонками, таблица (привязанная к шаблону таблицы), или разделитель. | `prisma.DocBlock` |
| **Шаблон таблицы** | Конфигурация колонок для табличных блоков (название таблицы БД → поле → ширина → порядок). | `prisma.TableTemplate` |
| **Тип документа** | Классификация шаблонов (КП, договор, счёт, акт…). | `prisma.DocType` |

## 👥 Роли и доступ

| Термин | Определение |
|--------|-------------|
| **admin** | Полный доступ ко всем разделам. |
| **manager** | Продажи, справочники (кроме админских), КП, договоры. |
| **production** | Производство, заказы, Гантт. |
| **storekeeper** | Склад, закупки. |
| **accountant** | Финансы. |
| **viewer** | Только просмотр. |

(Все роли — в `prisma.User.role`; доступ контролируется middleware `src/middleware.ts`.)

## 🔧 Технические термины

| Термин | Определение | Где |
|--------|-------------|------|
| **ADR** | Architecture Decision Record. Запись архитектурного решения в `docs/decisions/ADR-NNN-{name}.md`. | [docs/decisions/](./decisions/) |
| **Tier A (stable)** | Модуль в `STABLE-MODULES.md`, помечен как стабильный. **Запрещено** переписывать без ADR. | [`docs/operations/STABLE-MODULES.md`](./operations/STABLE-MODULES.md) |
| **Cycle** | Итерация разработки. Каждый cycle = 1 тематический commit + обновление `audit-log.md` + `CURRENT-CHECKLIST.md`. | [`docs/audit-log.md`](./audit-log.md) |
| **Скрам-фикс** | (Внутреннее) неологизм cycle v3.1.1: переход с `PrismaPg-only` на dual-adapter для устранения `SASL SCRAM-SERVER-FIRST-MESSAGE` ошибок при нестабильном Postgres. | `src/lib/db.ts`, `prisma/schema.prisma` |
| **Counter** | Автоинкремент номеров документов (КП-2026-0001, Д-2026-0001, etc.). Одна запись на тип документа в `prisma.Counter`. | `src/lib/counter.ts` |
| **Activity log** | Лог действий пользователя для аудита (кто создал/изменил/удалил, когда). | `src/lib/activity-log.ts` |
| **Rate limit** | Защита API от перегрузки (лимит запросов на IP/пользователя за окно). | `src/lib/rate-limit.ts` |
| **PRD** | Персональная наценка клиента (Personal Markup Percent). Поле в Proposal. | `prisma.Proposal.clientMarkupPercent` |
| **NDS (НДС)** | Налог на добавленную стоимость. Поле `vatRate` в Proposal. | `prisma.Proposal.vatRate` |
