## 🔍 Как искать здесь

Если ты ищешь **«почему было принято такое-то решение»** — архив планов (`plans/`) часто содержит контекст и альтернативы, которые НЕ вошли в итоговый ADR. Стоит заглянуть.

Если ты ищешь **«что было в старой версии SPEC»** — читай [`SPEC-ДОРАБОТКА.md`](./SPEC-ДОРАБОТКА.md); он хранит историю перехода v4.0 → v5.0.

Если ты ищешь **«что было закрыто как resolved»** — [`questions-2026.md`](./questions-2026.md) содержит стартовый/конечный статус каждого вопроса с указанием цикла.

## ⚠️ Правила архивации

1. **Никогда не редактируй архивные файлы** (даже для исправления опечаток): это исторические документы с зафиксированным состоянием.
2. **Новые решения на основе архива → создавай НОВЫЙ документ** в активной части (`docs/decisions/ADR-NNN-*.md`).
3. **Если ты считаешь, что архивный файл должен быть «реанимирован»** — создай issue / обсуждение, пересоздай документ в активной части, и оставь архивный как есть.
4. **Архив НЕ удаляется массово**. Даже самые стейл-документы могут содержать контекстные ссылки, которые невозможно восстановить.

---

**Поддерживай актуальность**: при архивации нового документа — добавь строку в таблицу выше (файл + дата + причина).
