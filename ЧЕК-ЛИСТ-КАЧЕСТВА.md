# ЧЕК-ЛИСТ-КАЧЕСТВА: Стандарт проверки модулей

> Каждый модуль проходит этот чек-лист перед тем как считаться завершённым.
> Провал любого пункта → [REWORK]

---

## 🔴 Критические (блокирующие)

- [ ] **TypeScript** — `npx tsc --noEmit` → 0 ошибок
- [ ] **Сборка** — `npm run build` (если возможно) → без ошибок
- [ ] **Нет dead code** — все импорты используются, нет `console.log`
- [ ] **Работоспособность в браузере** — страница открывается без 500 ошибок
- [ ] **API не падает** — запросы возвращают 200/201, а не 500

## 🟡 Обязательные

- [ ] **Loading state** — при загрузке данных есть spinner / skeleton
- [ ] **Empty state** — когда данных нет, показано понятное сообщение
- [ ] **Error state** — при ошибке API показано уведомление, а не белый экран
- [ ] **Edge cases** — пустая форма, удаление последнего элемента, 0 значений
- [ ] **Обработка сабмита** — кнопка блокируется при сохранении, показывает spinner
- [ ] **Валидация** — обязательные поля проверяются перед отправкой

## 🟢 Желательные

- [ ] **Адаптивность** — не ломается на мобильных (320px+)
- [ ] **Автосохранение** — черновик сохраняется в localStorage (если применимо)
- [ ] **Undo/Redo** — история изменений (если применимо)
- [ ] **Горячие клавиши** — Ctrl+Z, Ctrl+S (если применимо)
- [ ] **Тесты** — unit test на критическую логику
