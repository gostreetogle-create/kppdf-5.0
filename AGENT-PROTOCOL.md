# Agent Communication Protocol v3

## Как это работает

Вместо того чтобы писать друг другу в `AI_COLLABORATION.md`, агенты используют:

1. **`agent-queue.json`** — машиночитаемая очередь задач с чёткими статусами
2. **`agent-cli.js`** — CLI-утилита для управления очередью
3. **Сигналы** — асинхронные уведомления от одного агента другому

## ⚠️ ОБЯЗАТЕЛЬНО ПЕРЕД ЛЮБОЙ РАБОТОЙ

**ПЕРВАЯ КОМАНДА ПРИ СТАРТЕ СЕССИИ:**

```bash
node agent-cli.js <agent> check
```

**Если не сделаешь — пропустишь задачи от другого агента!**

Эта команда покажет:
- 🚨 Срочные сигналы (если есть — НЕМЕДЛЕННО реагируй!)
- 📨 Непрочитанные сигналы (ПРОЧИТАЙ ВСЕ!)
- 🎯 Задачи в работе
- 📋 Готовые к взятию задачи
- ➡️ Что делать ДАЛЬШЕ

**ПРАВИЛО: Никогда не начинай работу без проверки сигналов!**

## Статусы задач

| Статус | Значение | Команда |
|--------|----------|---------|
| `pending` | Готова к выполнению | (по умолчанию) |
| `in_progress` | В работе | `take <id>` |
| `done` | Выполнена | `done <id>` |
| `blocked` | Заблокирована | `block <id> <причина>` |
| `cancelled` | Отменена | (редактировать JSON) |

## Приоритеты сигналов

| Тип | Как отправить | Когда использовать |
|-----|---------------|-------------------|
| Обычный | `signal buffy "Текст"` | Информация, обновления |
| Срочный | `signal buffy "⚠️ СРОЧНО: ..."` | Нужна немедленная реакция |
| Блокер | `signal buffy "🚫 БЛОК: ..."` | Задача заблокирована, нужна помощь |

## Поток работы

```
Агент B начинает сессию:
  $ node agent-cli.js mimo check
  🔍 ПРОВЕРКА СОСТОЯНИЯ: MiMo Code Agent
  ========================================
  🚨 СРОЧНЫХ СИГНАЛОВ: 1
     ⚠️ [sig-005] от buffy: СРОЧНО: Возьми error-boundaries!
  📨 Непрочитанных сигналов: 3
  🎯 ГОТОВЫ К ВЗЯТИЮ (2):
     ⏳ error-boundaries (P1)
  ────────────────────────────────────────
  🚨 ПРИОРИТЕТ: ПРОЧИТАЙ СРОЧНЫЕ СИГНАЛЫ!

Агент B берёт задачу:
  $ node agent-cli.js mimo take error-boundaries

Агент B завершает задачу:
  $ node agent-cli.js mimo done error-boundaries
  ✅ Задача завершена!
  🎯 Освободились задачи:
     → dashboard-charts [MiMo]: Графики

Агент B сигналит агенту A:
  $ node agent-cli.js mimo signal buffy "error-boundaries готово!"
  📨 Сигнал отправлен Buffy (Codebuff)

Агент A проверяет сигналы:
  $ node agent-cli.js buffy signals
  📨 Сигналы (1):
     [sig-006] 📨 🆕 НОВОЕ
     От: MiMo → Кому: Buffy
     error-boundaries готово!

Агент A подтверждает сигнал:
  $ node agent-cli.js buffy ack sig-006
```

## Правила

1. **НАЧИНАЙ С `check`** — `node agent-cli.js <agent> check` — каждая сессия
2. **Бери задачу** — `take <id>` — перед тем как начать работать
3. **Завершай задачу** — `done <id>` — когда всё готово
4. **Не бери чужую задачу** — CLI проверит, что задача назначена на тебя
5. **Зависимости проверяются автоматически** — CLI не даст взять задачу, если её зависимости не выполнены
6. **Сигналы — для срочного** — когда нужно что-то сказать другому агенту
7. **Не пиши в AI_COLLABORATION.md** — используй очередь и сигналы
8. **Проверяй сигналы при старте** — `check` — не игнорируй их
9. **Отвечай на срочные сигналы НЕМЕДЛЕННО** — `urgent` и `blocker` требуют реакции

## Валидация

Периодически проверяй целостность очереди:

```bash
node agent-cli.js buffy validate
```

## Когда добавлять новую задачу

1. Не добавляй задачи вручную — напиши сигнал Buffy с описанием
2. Buffy обновит очередь с правильными зависимостями и приоритетами
3. Или редактируй `agent-queue.json` напрямую, если уверен

## Структура agent-queue.json

```json
{
  "version": 2,
  "last_updated": "2026-06-15T19:00:00.000Z",
  "agents": { "buffy": {...}, "mimo": {...} },
  "tasks": [
    {
      "id": "unique-task-id",
      "title": "Название задачи",
      "assignee": "buffy|mimo",
      "status": "pending|in_progress|done|blocked",
      "priority": 1-5,
      "depends_on": ["other-task-id"],
      "description": "Описание задачи"
    }
  ],
  "signals": [
    {
      "id": "sig-001",
      "from": "buffy",
      "to": "mimo",
      "type": "info|urgent|blocker",
      "message": "Текст сообщения",
      "acknowledged": false
    }
  ]
}
```
